# 仓库使用指南

欢迎加入「小登成长记录仓库」！本指南将帮助你正确使用仓库记录自己的学习历程，包括 Git 基础操作、目录规范、提交要求等内容。

# 一、前期准备
注册 Gitee 账号

若未注册，先进入 Gitee 官网 注册账号（建议用真实姓名作为用户名，方便识别）。

获取仓库权限

微信群里示意，提供你的 Gitee 用户名，申请加入仓库。管理员通过后，你会收到 Gitee 系统通知。

安装 Git 工具

下载地址：Git 官网（根据系统选择 Windows/macOS 版本）

安装时按默认选项即可，安装完成后，右键点击桌面会出现「Git Bash Here」（Windows）或通过终端（macOS）使用。

# 二、仓库基础操作（核心步骤）
## 1. 首次获取仓库到本地
打开 Git Bash（或终端），执行以下命令：

### 克隆仓库到本地（将仓库地址替换为实际地址）
git clone https://github.com/gududegoudaner/xiaodengsrecord.git

### 进入仓库目录（克隆后自动创建的文件夹）
cd xiaodengsrecord/42届电协
## 2. 创建个人目录
在仓库目录中，为自己创建独立文件夹（必须按规范命名）：

### 格式：姓名（如「张三」）
mkdir 张三

### 进入个人目录
cd 张三

## 3. 提交自己的内容
在个人目录中添加文件（如学习笔记、代码等）后，执行以下步骤提交到远程仓库：

### 回到仓库根目录（若已在根目录可跳过）
cd ..

### 查看当前修改状态（可选，确认要提交的文件）
git status

### 将所有修改添加到暂存区（. 表示所有文件，也可指定具体文件名）
git add .

### 提交修改（填写有意义的备注，说明本次提交的内容）
git commit -m "张三：添加C语言基础学习笔记"

### 推送到远程仓库（首次推送可能需要输入Gitee账号密码，或验证令牌）
git push
## 4. 同步仓库最新内容
由于多人协作，其他人可能已提交新内容，每次修改前建议先同步：

### 拉取远程仓库的最新版本
git pull
# 三、目录与内容规范
## 1. 个人目录结构（建议）
在自己的「姓名 - 学号」目录下，建议按以下结构组织内容（可根据需求调整）：

plaintext
张三-2023001/
├── README.md           # 个人简介（必填）：包含姓名、学号、学习目标、擅长技术等
├── 学习笔记/            # 按时间或主题分类（如「2023-10-Python基础」「2023-11-Web开发」）
│   ├── 笔记1.md
│   └── 代码示例/        # 笔记中涉及的代码片段
├── 项目代码/            # 完整项目（如课程作业、个人小项目）
│   ├── 项目1/           # 每个项目单独建文件夹
│   │   ├── 代码/
│   │   └── 说明.md      # 项目功能、运行方法等
├── 技术总结/            # 对知识点的梳理（如「Python函数总结.md」「数据库优化技巧.md」）
└── 资源链接.md          # 收藏的学习网站、工具、教程等
## 2. 内容格式要求
文件命名：用中文或英文，避免特殊符号（如空格、问号），建议用「日期 - 名称」格式（如「20231001-Python 变量.md」）。
笔记格式：推荐用 Markdown（.md 文件），方便在线预览（Gitee 支持 Markdown 渲染）。
代码文件：保留完整代码和注释，方便他人理解。
禁止内容：不允许上传大文件（如超过 100MB 的安装包、视频）、无关文件（如游戏、私人照片）、侵权内容。
# 四、常见问题解决
提交时提示「冲突」
原因：多人修改了同一文件，导致 Git 无法自动合并。
解决：
先执行 git pull 拉取最新版本，手动修改冲突部分（文件中会标记 <<<<<<< HEAD 等冲突标记）。
修改后重新执行 git add . → git commit -m "解决冲突" → git push。
忘记 Git 命令
可参考 Git 常用命令速查表 或在 Git Bash 中输入 git --help 查看帮助。
无法 push 推送
检查：
是否已加入仓库（联系管理员确认权限）。
账号密码是否正确（若用令牌验证，需在 Gitee 「设置 - 安全设置 - 私人令牌」中生成）。
网络是否正常，尝试重新执行 git push。
